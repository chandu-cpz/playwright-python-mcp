# playwright-python-mcp

Python-native port of the official Playwright MCP server.

This package gives MCP clients browser automation tools backed by Python Playwright. It follows the official Playwright MCP public contract: tool names, capability names, schemas, snapshot/ref behavior, and browser-side semantics are kept aligned with upstream.

The intentional divergence is code generation language. Upstream emits JavaScript snippets; this port emits Python Playwright snippets:

```python
await page.get_by_role("button", name="Submit").click()
```

## Status

- Package version: `0.0.1`
- Python: `>=3.12`
- MCP adapter: FastMCP
- Browser runtime: Python Playwright
- License: Apache-2.0
- Tested Python versions: 3.12 locally; package metadata also supports 3.13

FastMCP is used only as the MCP transport/tool adapter layer. Browser behavior lives in normal Python backend modules so the MCP framework remains replaceable.

## Install

Run from PyPI with `uvx`:

```bash
uvx playwright-python-mcp --headless
```

Run from this checkout:

```bash
uv run playwright-python-mcp --headless
```

Run as a Python module:

```bash
uv run python -m playwright_python_mcp --headless
```

Install browser binaries when needed:

```bash
uv run playwright-python-mcp install-browser chromium
```

## MCP Client Config

Most MCP clients can use stdio:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "uvx",
      "args": ["playwright-python-mcp", "--headless"]
    }
  }
}
```

For opencode:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "playwright": {
      "type": "local",
      "command": ["uvx", "playwright-python-mcp", "--headless"],
      "enabled": true
    }
  }
}
```

For Codex:

```toml
[mcp_servers.playwright]
command = "uvx"
args = ["playwright-python-mcp", "--headless"]
```

## Transport

By default the server uses stdio. Passing `--port` starts Streamable HTTP at `/mcp`:

```bash
uvx playwright-python-mcp --headless --host 127.0.0.1 --port 8931
```

Client config for HTTP transport:

```json
{
  "mcpServers": {
    "playwright": {
      "url": "http://127.0.0.1:8931/mcp"
    }
  }
}
```

## Common Options

```bash
uvx playwright-python-mcp \
  --browser=chromium \
  --headless \
  --caps=vision,pdf,storage,testing,config \
  --output-dir=.playwright-mcp \
  --snapshot-mode=full
```

Useful flags:

| Option | Purpose |
| --- | --- |
| `--browser` | Browser engine or channel. |
| `--headless` | Run browser headlessly. |
| `--caps` | Enable additional capability families such as `vision`, `pdf`, `storage`, `testing`, `config`, `network`, `tracing`, or `devtools`. |
| `--cdp-endpoint` | Connect to an existing Chromium browser over CDP. |
| `--endpoint` | Connect to a bound Playwright browser endpoint. |
| `--extension` | Connect through the Playwright browser extension relay. |
| `--isolated` | Use an isolated browser context instead of a persistent profile. |
| `--storage-state` | Load storage state from a JSON file. |
| `--output-dir` | Directory for files generated by tools. |
| `--output-mode` | Return generated artifacts inline or by file reference. |
| `--snapshot-mode` | Control automatic snapshot output: `full` or `none`. |
| `--test-id-attribute` | Attribute used for test-id locators. |
| `--timeout-action` | Default Playwright action timeout in milliseconds. |
| `--timeout-navigation` | Default navigation timeout in milliseconds. |

Run `playwright-python-mcp --help` for the complete option list.

## Browser Profiles And Connections

By default, the server launches a persistent browser context using an auto-created profile under the user cache directory, matching upstream Playwright MCP session behavior.

Explicit persistent profile:

```bash
uvx playwright-python-mcp --user-data-dir ~/.cache/playwright-python-mcp/profile
```

Isolated session:

```bash
uvx playwright-python-mcp --isolated --storage-state auth-state.json
```

`--isolated` uses an ephemeral browser and cannot be combined with `--user-data-dir`.

Existing browser over CDP:

```bash
uvx playwright-python-mcp --cdp-endpoint http://localhost:9222
```

Bound Playwright browser endpoint:

```bash
uvx playwright-python-mcp --endpoint ws://localhost:3000
```

Browser extension mode:

```bash
uvx --from 'playwright-python-mcp[extension]' playwright-python-mcp --extension
```

`--extension` starts a local Python CDP relay and opens the Playwright browser extension connect page. The Playwright Extension must already be installed in the selected Chrome or Edge profile. The relay dependency is optional, so published installs need the `extension` extra.

## Configuration

Configuration is merged in this order:

1. Built-in defaults.
2. JSON or INI file from `--config` or `PLAYWRIGHT_MCP_CONFIG`.
3. `PLAYWRIGHT_MCP_*` environment variables.
4. CLI flags.

Example:

```bash
PLAYWRIGHT_MCP_TIMEOUT_NAVIGATION=45000 \
uvx playwright-python-mcp --config playwright-mcp.ini --timeout-action 10000
```

INI example:

```ini
capabilities = config,storage
console.level = error
timeouts.action = 10000
timeouts.navigation = 45000
browser.contextOptions.viewport = 1280x720
```

## Known Limitations

- `browser_annotate` is intentionally not exposed. Upstream implements it through the Node.js Playwright Dashboard daemon; this port does not ship that daemon. A Python-native equivalent may be added later if the dashboard workflow is ported.
- `browser.initPage` / `--init-page` is parsed for config compatibility but not executed. Upstream loads a JavaScript module and calls its default export against each new page; this Python package does not execute Node.js init-page modules.
- HTTP transport currently uses one backend instance for the server process. This gives shared browser state, but it is not yet the full upstream per-client backend factory model.
- HTTP transport does not yet implement upstream's periodic MCP heartbeat ping or `/killkillkill` shutdown endpoint. Stdio transport has signal cleanup.
- A small number of conformance tests are explicitly skipped for fixture or environment gaps, including bound-browser debugger closure, remote endpoint storage-state coverage, CDP first-tab reuse coverage, and record-video coverage.

## Security

Playwright MCP is not a security boundary. It gives an MCP client the ability to drive a browser and interact with local files allowed by configuration. Use client-level permissions, workspace restrictions, and network controls when exposing it to agents.
